# D0.9 — 공유 데이터 계약 게이트 문서

> M0 exit gate. `CONTRACT.md`를 게이트 문서로 정리한 요약본.  
> 변경은 `CONTRACT.md` ADR 개정을 통해서만 가능하다.

## CONTRACT.md 핵심 요약

### (i) 안정키 ↔ 카탈로그/매니페스트 필드 매핑

| 필드 | 위치 | 역할 |
|------|------|------|
| `element.key` | 매니페스트 | 안정 식별자 (인스턴스) |
| `QmManaged.stableKey` | Unity 프리팹 | `element.key`와 1:1 매핑 |
| `component` ID | 카탈로그 키 | 부품 타입 식별자 |

**계약**: `manifest.element.key === QmManaged.stableKey`

key 형식 정규식: `^[a-z][a-z0-9]*(\.[a-z][a-z0-9]*)*$`

예시: `panel.roster`, `list.mercs`, `btn.recruit`

### (ii) 정규화 단계 위치

데이터 흐름:
```
manifest(JSON) → merge-planner → [프리팹 YAML 읽기 → normalize → diff → 계획] → MCP 적용기
```

- 정규화(`normalize-prefab.mjs`)는 **출력 단계**에만 적용
- 매니페스트 JSON에는 정규화 미적용
- 트랙 A(매니페스트)는 정규화를 모른다

### (iii) 골든 픽스처 소유권

| 디렉토리 | 소유 트랙 | 용도 |
|---------|---------|------|
| `test/fixtures/manifests/` | 트랙 A | 매니페스트 검증 픽스처 |
| `test/fixtures/golden/` | 트랙 B | 프리팹 정규화 골든 |
| `test/e2e/GS1, GS2` | 공유 (M4 후) | e2e 보존/무손실 |

### 검증 계약 (D0.2)

- L2/L3 모두 `runtime/validate-manifest.mjs` 단일 함수 호출 (SSOT)
- 반환: `{ ok: boolean, errors: [{ code, key, message }] }`
- exit code: 유효=0, 무효=2
- 거부 코드: `SCHEMA_VIOLATION` / `CATALOG_UNKNOWN_COMPONENT` / `DUPLICATE_KEY` / `MISSING_REQUIRED` / `OVERRIDABLE_VIOLATION` / `INVALID_KEY_FORMAT`

### 런타임 계약

- 모든 `runtime/*.mjs`: ESM, Node ≥18
- 순수 함수: default export + named export
- hook 호출 형식: `node "${CLAUDE_PLUGIN_ROOT}/runtime/<file>.mjs"`

