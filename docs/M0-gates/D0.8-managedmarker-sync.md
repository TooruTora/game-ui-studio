# D0.8 — ManagedMarker 동기화 모델

> M0 gate 문서. GS2와 연결.  
> **Unity 환경 필요**: 동기화 구현 및 검증은 Unity Editor가 필요하다.  
> 현재 상태: 설계 완료, 구현 미완 (M5 예정).

---

## 개요

Game UI Studio가 생성/관리하는 파일을 사용자가 수정한 후 플러그인을 업그레이드/재설치할 때 수정 내용을 무손실로 보존하는 동기화 모델.

---

## 경로 결정

### 문제: Unity의 `Assets/` 제약

Unity Editor는 **`Assets/` 하위 파일만 AssetDatabase로 인식**한다.

- `${CLAUDE_PLUGIN_DATA}/game-ui-studio/` — Claude Code 플러그인 데이터 영역 (Unity 미인식)
- `Assets/GameUiStudio/` — Unity 인식 영역 (프리팹, 카탈로그 사본)

이 제약으로 인해 Game UI Studio는 두 위치를 동시에 관리해야 한다.

### 경로 결정 테이블

| 파일 유형 | 권한 출처 | 경로 | 이유 |
|-----------|----------|------|------|
| 매니페스트 JSON | Claude Code | `${CLAUDE_PLUGIN_DATA}/game-ui-studio/manifests/` | Unity 불필요 |
| 카탈로그 JSON | Claude Code (정본) | `${CLAUDE_PLUGIN_DATA}/game-ui-studio/catalog.json` | Unity 불필요 |
| 카탈로그 사본 | 동기화 | `Assets/GameUiStudio/catalog.json` | Unity 런타임 필요 |
| 생성 프리팹 | Unity | `Assets/UI/Prefabs/*.prefab` | Unity AssetDatabase |
| 해시 레코드 | Claude Code | `${CLAUDE_PLUGIN_DATA}/game-ui-studio/.hashes.json` | 동기화 판정용 |

> **명문화**: 프리팹 파일은 반드시 `Assets/` 하위에 있어야 한다. `${CLAUDE_PLUGIN_DATA}`에 프리팹을 저장하면 Unity Editor가 임포트하지 못한다. 단, 매니페스트/카탈로그 정본은 `${CLAUDE_PLUGIN_DATA}`에 두고 `Assets/GameUiStudio/`에 사본을 동기화하는 이중 경로 방식을 취한다.

## 콘텐츠 해시 + 사용자 수정 3-way 결정표

### 해시 레코드 구조

```json
{
  "version": "0.3.0",
  "files": {
    "Assets/GameUiStudio/catalog.json": {
      "installedHash": "sha256:abc123...",
      "recordedAt": "2026-06-22T00:00:00Z"
    }
  }
}
```

`installedHash`: 플러그인이 해당 파일을 배포/생성했을 때의 SHA-256 해시.  
현재 파일 해시와 비교하여 사용자 수정 여부를 판정한다.

### 4상한 결정표

| 상한 | `installedHash == 현재해시` | 사용자 수정 | 신규버전 충돌 | 결정 | 동작 |
|------|--------------------------|------------|-------------|------|------|
| Q1 | 일치 | 없음 | N/A | 자동 덮어쓰기 | 신규 버전 적용, 해시 갱신 |
| Q2 | 일치 | 없음 | N/A | (Q1과 동일) | — |
| Q3 | 불일치 | 있음 | 없음 | 사용자 수정 유지 | 신규 버전 비충돌 변경만 병합 |
| Q4 | 불일치 | 있음 | 있음 | 사용자 선택 | 3-way diff UI 표시, 대기 |

> Q2는 이론적으로 발생하지 않는다 (해시가 일치하면 수정이 없음).

### 상한별 상세 동작

#### Q1 — 자동 덮어쓰기 (안전)

```
조건: currentHash(file) === hashes[file].installedHash
동작:
  1. 신규 버전 파일로 덮어쓰기
  2. hashes[file].installedHash = hash(newFile)
  3. hashes[file].recordedAt 갱신
사용자 알림: "파일이 v{version}으로 업데이트되었습니다"
```

#### Q3 — 사용자 수정 유지 (비충돌 병합)

```
조건: currentHash(file) !== hashes[file].installedHash
      AND 신규 버전이 동일 영역 미수정
동작:
  1. diff3(사용자수정본, 설치본, 신규버전) 계산
  2. 비충돌 영역: 신규 버전 변경 적용
  3. 사용자 수정 영역: 보존
  4. 결과 저장, 해시 갱신
사용자 알림: "사용자 수정을 보존하며 v{version} 변경을 병합했습니다"
```

#### Q4 — 사용자 선택 (충돌)

```
조건: currentHash(file) !== hashes[file].installedHash
      AND 신규 버전이 동일 영역 수정 (충돌)
동작:
  1. 충돌 감지, 파일 미변경
  2. Claude Code UI에 표시:
     - [내 수정 유지] → 사용자 버전 유지, 신규 버전 무시
     - [신규 버전 적용] → 신규 버전으로 덮어쓰기
     - [수동 편집] → diff3 결과를 편집기로 오픈
  3. 사용자 선택 후 해시 갱신
```
